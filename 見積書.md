# 楽天RMSレポート自動収集・統合基盤システム 御見積書

**作成日**: 2024年12月（最新更新）  
**プロジェクト名**: 楽天RMS RPPレポート自動収集・統合基盤構築（OAuth2認証 / n8n連携対応）  
**対象**: 複数店舗管理・拡張性を前提とした、楽天RMSデータ取得APIシステムの構築

-----

## 1\. 導入のメリットと本システムの価値

本システムは、単なるデータダウンロードツールではなく、御社の事業拡大を支える\*\*「自社データインフラ」\*\*として設計されています。

### 既存SaaS（Teps等）との比較

| 項目 | 一般的なSaaSツール | **本システム（自社基盤）** |
| :--- | :--- | :--- |
| **費用構造** | アカウント数（店舗数）ごとの課金。<br>クライアントが増えるほどコスト増。 | **店舗数無制限**。<br>クライアントが100社に増えてもシステム費用は一定。 |
| **データ活用** | 画面で見るのみ、または連携先が限定的。 | **完全自由**。<br>生データ(CSV)を自社DB、BigQuery、BIツールへ統合可能。 |
| **拡張性** | 機能追加はベンダー次第。 | **OAuth2 / n8n連携**により、独自の自動入札ロジックやAI解析へ拡張可能。 |

## 3\. 開発詳細仕様

### 3.1 機能一覧

#### コア機能（データ収集）

  - **FastAPI / Playwright**: 高速かつ安定したブラウザ自動操作によるレポート取得
  - **Dockerコンテナ化**: 環境依存を排除し、将来的なサーバー移転やスケールアウトも容易
  - **エラー自動回復**: ネットワーク瞬断やタイムアウト時の自動リトライ機能

#### 拡張・連携機能（OAuth2実装）

  - **OAuth2認証サーバー**: Client Credentialsフロー対応（n8n等のiPaaSとセキュアに連携）
  - **JWTトークン管理**: リフレッシュトークン対応、ステートレスな認証基盤
  - **セキュリティ**: パスワードのハッシュ化（bcrypt）、環境変数による機密情報管理

### 3.2 納品物

  - ソースコード一式（Python / FastAPI）
  - Docker構成ファイル（Dockerfile, docker-compose.yml）
  - API仕様書・連携マニュアル（README, N8N\_SETUP.md等）
